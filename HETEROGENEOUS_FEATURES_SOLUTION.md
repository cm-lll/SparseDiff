# 异质图特征嵌入解决方案

## 问题分析

当前的 `EigenFeatures` 是为**同质图**设计的：
- 将所有边类型合并：`A = E_t[..., 1:].sum(dim=-1)`
- 计算单一的拉普拉斯矩阵
- **忽略了异质图的节点类型和边类型（关系族）信息**

对于异质图，这种处理方式：
1. **丢失了重要的结构信息**：不同关系族的结构特征被混合
2. **数值稳定性问题**：合并后的图可能有更多连通分量和重复特征值
3. **不适合异质图**：异质图需要考虑节点类型和边类型的异质性

## 解决方案

### 方案1：禁用特征值特征（推荐，最简单）

在配置文件中设置：
```yaml
model:
  eigenfeatures: False  # 禁用特征值特征
```

**优点**：
- 简单直接
- 避免数值稳定性问题
- 其他特征（邻接特征、度分布等）仍然可用

### 方案2：改进现有实现（已实现）

已修改 `ExtraFeatures` 类：
1. **自动检测异质图**：通过 `dataset_info.heterogeneous` 判断
2. **添加警告**：如果是异质图且使用特征值特征，会给出警告
3. **错误处理**：如果特征值计算失败，使用零特征代替，避免训练中断

**优点**：
- 向后兼容
- 自动处理错误
- 给出明确警告

### 方案3：创建专门的异质图特征类（未来改进）

可以创建 `HeterogeneousEigenFeatures` 类，考虑：
1. **按关系族计算特征**：为每个关系族计算独立的拉普拉斯矩阵
2. **节点类型统计**：不同节点类型的度分布
3. **关系族统计**：每个关系族的边数和连通性

## 当前建议

**对于异质图，建议禁用特征值特征**：
```yaml
model:
  eigenfeatures: False
```

其他特征（如 `AdjacencyFeatures`）仍然可以正常工作，因为它们不依赖于将所有边类型合并。
