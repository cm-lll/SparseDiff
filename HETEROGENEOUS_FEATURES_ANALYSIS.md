# 异质图特征嵌入分析

## 当前实现的问题

### 1. EigenFeatures 是为同质图设计的

当前的 `EigenFeatures.compute_features()` 方法：
```python
A = E_t[..., 1:].sum(dim=-1).float()  # 将所有边类型合并
L = self.compute_laplacian(A, normalize=False)  # 计算单一拉普拉斯矩阵
```

**问题**：
- 将所有边类型（关系族）合并成一个邻接矩阵
- 忽略了异质图的节点类型和边类型信息
- 对于异质图，这种处理方式丢失了重要的结构信息

### 2. 异质图应该考虑的特征

对于异质图，应该考虑：

1. **按关系族的邻接矩阵**：
   - 每个关系族有独立的邻接矩阵
   - 可以计算每个关系族的拉普拉斯矩阵特征值

2. **节点类型分布**：
   - 不同节点类型的数量
   - 节点类型的度分布

3. **边类型（关系族）分布**：
   - 每个关系族的边数
   - 关系族的连通性

4. **元路径特征**（可选）：
   - 例如：Author -> Paper -> Author
   - 可以捕获异质图中的高阶关系

## 解决方案

### 方案1：禁用特征值特征（临时方案）
```yaml
model:
  eigenfeatures: False
```

### 方案2：创建异质图特征类（推荐）

创建一个新的 `HeterogeneousEigenFeatures` 类，考虑：
1. 按关系族计算特征值
2. 节点类型统计
3. 关系族统计

### 方案3：改进现有实现

修改 `EigenFeatures` 使其能够：
1. 检测是否为异质图
2. 如果是异质图，使用不同的特征计算方法
3. 或者为每个关系族单独计算特征
