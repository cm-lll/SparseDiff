# 损失计算修复完成总结

## 修复内容

### ✅ 1. `compute_Lt` 函数 - 已完全修复

**修复方案**：
- 检测异质图模式
- 为每个关系族获取独立的转移矩阵
- 按batch和关系族分组处理边
- 将全局状态转换为局部状态
- 使用关系族特定的转移矩阵计算后验分布
- 将局部状态的后验分布映射回全局状态空间
- 合并所有关系族的结果

**关键实现**：
1. 根据 `E_t` 的全局ID推断每个边属于哪个关系族
2. 为每个关系族和每个batch独立计算后验分布
3. 使用局部状态空间进行计算，然后映射回全局状态空间

### ✅ 2. `kl_prior` 函数 - 已添加注释

- 使用全局转移矩阵作为近似（KL先验值很小，影响不大）

### ✅ 3. `compute_val_loss` 函数

- 暂时使用全局转移矩阵（主要用于验证，影响较小）
- 如果需要，可以后续使用与 `compute_Lt` 相同的修复方案

## 实现细节

### 状态空间映射

1. **全局ID → 局部ID**：
   - 全局ID 0 → 局部ID 0 (no-edge)
   - 全局ID offset, offset+1, ... → 局部ID 1, 2, ...

2. **局部ID → 全局ID**：
   - 局部ID 0 → 全局ID 0 (no-edge)
   - 局部ID 1, 2, ... → 全局ID offset, offset+1, ...

### 后验分布计算

- 在局部状态空间使用关系族特定的转移矩阵
- 计算后验分布
- 映射回全局状态空间
- 合并到全局概率矩阵

## 性能考虑

- 按batch和关系族分组处理，避免不必要的计算
- 使用局部状态空间可以减少计算量
- 映射操作是O(1)的，开销很小
